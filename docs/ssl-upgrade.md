# SSL — Self-Signed to Let's Encrypt Migration Guide

The project currently uses a self-signed certificate generated by `nginx/generate-ssl.sh`.
This is appropriate for an IP-based deployment where Let's Encrypt is not available.

When a domain name is added, follow these steps to migrate to a trusted Let's Encrypt certificate.

---

## Why Self-Signed Now

- Let's Encrypt requires a valid public domain name (not an IP address)
- For this assignment, the EC2 public IP is used directly
- Self-signed certs provide the same encryption — browsers just show a warning
- The NGINX config is already structured for easy migration

---

## Migration Steps (When You Have a Domain)

### 1. Point Your Domain to EC2

Add an A record in your DNS provider:
```
A   yourdomain.com       <EC2-Public-IP>
A   www.yourdomain.com   <EC2-Public-IP>
```

Wait for DNS propagation (5–30 minutes).

### 2. Install Certbot on EC2

```bash
sudo apt update
sudo apt install -y certbot python3-certbot-nginx
```

### 3. Update nginx.conf

Change `server_name _;` to your actual domain in both server blocks:
```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    ...
}

server {
    listen 443 ssl;
    server_name yourdomain.com www.yourdomain.com;
    ...
}
```

Sync the updated nginx.conf to EC2 and reload:
```bash
docker compose exec nginx nginx -s reload
```

### 4. Obtain Certificate

Stop the nginx container temporarily (Certbot needs port 80):
```bash
docker compose stop nginx
sudo certbot certonly --standalone -d yourdomain.com -d www.yourdomain.com
docker compose start nginx
```

Or use the webroot method (nginx stays up):
```bash
sudo certbot certonly --webroot -w /var/www/certbot \
  -d yourdomain.com -d www.yourdomain.com
```

Certificates will be saved to:
```
/etc/letsencrypt/live/yourdomain.com/fullchain.pem
/etc/letsencrypt/live/yourdomain.com/privkey.pem
```

### 5. Update nginx.conf — Swap Certificate Paths

Replace the `ssl_certificate` lines:
```nginx
# Before (self-signed):
ssl_certificate     /etc/nginx/ssl/cert.pem;
ssl_certificate_key /etc/nginx/ssl/key.pem;

# After (Let's Encrypt):
ssl_certificate     /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
```

### 6. Mount Certificate Directory in docker-compose.yml

```yaml
nginx:
  volumes:
    - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
    - /etc/letsencrypt:/etc/letsencrypt:ro   # Add this line
```

Restart nginx:
```bash
docker compose up -d --no-deps nginx
```

### 7. Auto-Renewal

Certbot installs a systemd timer that renews automatically. Verify it:
```bash
sudo systemctl status certbot.timer
sudo certbot renew --dry-run
```

After renewal, reload nginx:
```bash
# Add to /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
#!/bin/bash
docker exec nginx_proxy nginx -s reload
```

## Verify

```bash
# Should show valid cert (no -k flag needed)
curl https://yourdomain.com/health

# Check cert details
openssl s_client -connect yourdomain.com:443 -brief
```
